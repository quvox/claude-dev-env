# プロダクト概要

## これは何か

`claude-dev-env` は、Claude Code を安全な Docker コンテナ内で動かす**開発環境一式**と、
その環境上で複数のコーディングエージェントを連携させる**AIオーケストレーター**を提供する
ツールキット。ホスト側 CLI（`claude-dev` / `claude-dev-mac`）から `docker run` 相当の操作一発で、
プロジェクトごとに隔離されたコンテナを起動して開発する。

## 誰の何を解決するか

- **対象ユーザー**: 社内の開発チーム（信頼できる社内開発用途に限定）。
- **課題**: 人手を増やせない中で並列プロジェクト数を上げたい。人間がオーケストレーター（進行管理）を
  担うと管理に時間を取られ、並列度が上がらない。またエージェント実行はホスト資産への到達リスクを伴う。
- **価値**:
  1. **隔離**: レビュー前コードをコンテナ境界内で実行し、ホスト（認証情報・FS）への到達経路を作らない。
  2. **再現性・配布**: 環境を Docker イメージとして配布し、チーム全員が同一構成を使う。
  3. **並列度**: プロジェクトごとに AIオーケストレーターを1体立て、人間は「介入トリガー該当時だけ」対話する。

## 中核となる考え方

- **単一コンテナ同居型**: root（オーケストレーター）とワーカー（Claude / Codex 等）は同一コンテナ・
  同一 FS を共有する。隔離境界はコンテナとホストの間にのみ置く（個別エージェント隔離はしない割り切り）。
- **人間の関与は例外時だけ**: 後戻り不能な重大判断・要件の曖昧さ・行き詰まり・方針分岐・前提崩れの
  5トリガー該当時のみ停止して人間に確認する。それ以外は仮定を記録して自律継続する。
- **異種エージェント混在**: 実装と独立レビューを別ベンダーのワーカーに担わせ、盲点を相互補完する。

## デザイン原則（UI）

UI はターミナル主体。ホスト CLI（`claude-dev`）のサブコマンド体系と、オーケストレーターの
TUI ダッシュボード（bubbletea/lipgloss）が主な接点。ブラウザ確認は noVNC 経由で提供する。
装飾より情報密度と操作の一貫性を優先する。
